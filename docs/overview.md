# ChainMove Overview

## Summary

ChainMove is an in-development mobility payments platform being adapted for Stellar. The product is designed to support ride booking, escrowed payment handling, driver payouts, and future driver financing flows using blockchain-based settlement.

## Product Vision

The long-term goal is to turn transport into a reliable on-chain payment category. Riders should be able to fund trips with confidence, drivers should receive transparent and timely settlement, and platform operators should have auditable payment rails.

## Target Users

- Riders who need transparent digital fare handling
- Drivers who need faster settlement and better cash flow
- Fleet operators or dispatch partners
- Sponsors, employers, or investors funding driver activity
- Ecosystem partners building wallets, anchor services, or compliance integrations

## Primary Use Cases

- Solo ride payments with escrowed release
- Shared ride coordination with pooled settlement
- Driver payout tracking and treasury accounting
- Stable asset-based trip settlement
- Sponsor-funded or employer-funded mobility credits

## Why It Fits Stellar

ChainMove benefits from low-cost payment rails, asset issuance support, and Soroban programmability. Stellar gives the project a practical path to consumer-friendly transaction flows without optimizing around speculative behavior.

## Current State

The repository contains an earlier decentralized transport prototype built with Next.js and a Motoko backend. The Stellar version described in this documentation is the proposed next implementation phase.
